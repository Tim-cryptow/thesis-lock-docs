The web app at [thesis-lock.vercel.app](https://thesis-lock.vercel.app) is the primary way to create anchors and explore them. Everything here reads from or writes to the same public contracts described in the [contracts reference](/reference/contracts/overview/).

## Connecting a wallet

Anchoring requires a Stacks wallet. ThesisLock uses Stacks Connect and supports [Leather](https://leather.io), [Xverse](https://www.xverse.app), and [Asigna](https://asigna.io). Connecting only authorizes signing; ThesisLock never takes custody of funds. Reading and verifying need no wallet at all.

## Anchoring a document

Drop a file onto the anchor area or pick one. Your browser computes the SHA-256 digest locally and displays it. Add an optional label (up to 64 ASCII characters), then sign the transaction. This calls `anchor-document` on the `thesislock` contract and records the signer, the Stacks and Bitcoin block heights, and the label. The same hash cannot be single-anchored twice; a duplicate is rejected on chain.

## Batch anchoring

When you have several files to timestamp together, batch anchoring records up to ten hashes in one transaction through the `thesislock-batch` contract. This saves fees compared with anchoring each file separately. Batch records are keyed by hash and owner, so to verify one later you supply the owner principal as well as the hash.

## Groups

Groups are shared, named timelines backed by the `thesislock-groups` contract. Create a group to become its admin and first member, then add members by principal. Any member can anchor a hash into the group's ordered ledger; the admin manages membership. Groups suit labs, teams, or collaborations that want a single collective record of artifacts.

## Proof NFTs

Minting a proof NFT through the `thesislock-proof` contract issues a SIP-009 token to your wallet that represents the anchor. The token is soulbound: it cannot be transferred and stays permanently with the minting wallet. Each token carries the hash, label, minter, and block heights, and can be looked up by hash or by token id.

## Verification and hash matching

Open `https://thesis-lock.vercel.app/v/<hash>` to see an anchor's owner, label, and block heights. To prove a specific file matches, use hash matching: re-upload the file on the verification page and the browser hashes it locally and compares. A match confirms the file is byte-for-byte identical to what was anchored. The upload is for local hashing only and is never transmitted.

## Profiles and wallet pages

Each wallet has a public profile showing its anchoring activity and stats, backed by the registry contract and the optional index. Profiles are shareable and give a readable history of a principal's anchors without exposing any document.

## Watchlist, collections, and favorites

The app includes organizational tools layered on top of the chain:

- Watchlist: follow wallets or anchors and track new activity.
- Collections: group anchors into your own curated sets for browsing.
- Favorites: bookmark anchors you return to often.

These views are conveniences for navigation. The authoritative records remain on chain, and anything shown can be reconfirmed against the contracts.

## Activity, search, and stats

The app surfaces a live activity stream, full-text-style search over hashes, addresses, and labels, and protocol statistics. These read from the optional index for speed, as explained in [On-chain truth vs the index](/concepts/on-chain-vs-index/). Newly mined anchors are verifiable immediately and appear in these discovery views shortly after, once the chainhook delivers them.

## Sharing

Anchors can be shared with badge and card images generated by the [REST API](/reference/rest-api/): an SVG badge for embedding in a README, and an Open Graph card for link previews. Both are addressable by hash.
